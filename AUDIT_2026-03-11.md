## 7) Zones de confusion qui ont causé du flou (et correctifs)

1. **Docs/Slides**: indisponibles officiellement côté Synology (retour support), donc non implémentables proprement aujourd’hui.
2. **SynologyOffice vs SynologySheets**: risque de confusion fonctionnelle.
   - Reco: considérer `SynologySheets` comme node principal tableur.
   - Garder `SynologyOffice` pour discovery/fallback/custom Office call.
3. **`REPORT.md` pas à jour** vs versions réellement utilisées (0.31.20 / 0.31.21).

---

## 8) Priorités recommandées (ordre concret)

1. **Mettre à jour `REPORT.md`** avec l’état réel (versions, tests, blocages).
2. **Tagger clairement les nodes “primary path”**:
   - Sheets = primaire tableur
   - Office = fallback/générique
3. **Ajouter matrice de validation** par opération:
   - `OK` / `PARTIAL` / `KO` / `NOT_TESTED`
4. **Monter une vraie suite tests** (au moins smoke par node + 5 cas critiques/node).
5. **Stabiliser release discipline**: commit + changelog + publish checklist.
